Enhance the Django tutor management system to fully support bilingual functionality (Arabic and English) across the entire application.

---

## MULTI-LANGUAGE SUPPORT (CRITICAL)

The system must support:

* Arabic (ar)
* English (en)

ALL UI text must be translatable.

---

1. FULL DJANGO I18N IMPLEMENTATION

---

* Enable Django internationalization (i18n)

* Configure:

  * LANGUAGES = [("en", "English"), ("ar", "Arabic")]
  * USE_I18N = True
  * LOCALE_PATHS

* Use:

  * gettext / gettext_lazy for ALL strings

* No hardcoded text is allowed in:

  * templates
  * views
  * forms
  * validation messages

---

2. LANGUAGE SWITCHING

---

* Add language switcher in UI:

  * Arabic / English toggle

* Store user preference:

  * session or user profile

* Automatically apply selected language across the system

---

3. TEMPLATE TRANSLATION

---

* Use Django template tags:

  * {% trans %}
  * {% blocktrans %}

* Translate ALL:

  * buttons
  * labels
  * messages
  * headings
  * alerts

---

4. RTL / LTR SUPPORT (IMPORTANT)

---

* Arabic:

  * direction: RTL

* English:

  * direction: LTR

* Dynamically adjust:

  * layout
  * alignment
  * navigation

* Use CSS:

  * direction: rtl / ltr
  * text-align adjustments

---

5. FORM & VALIDATION MESSAGES

---

* Translate:

  * form labels
  * error messages
  * validation messages

---

6. DATABASE CONTENT RULE (IMPORTANT)

---

* DO NOT translate user-entered data:

  * student names
  * notes
  * custom text

* Data must remain:

  * exactly as entered by user

---

7. DATE & TIME LOCALIZATION

---

* Format:

  * dates
  * times

based on selected language

---

8. INTEGRATION WITH EXISTING FEATURES

---

Ensure translation works correctly with:

* Scheduling system
* Timezone display (dual time UI)
* Prayer time messages
* Dashboard
* Analytics charts (labels, legends)

---

9. TRANSLATION FILES

---

* Generate:

  * .po files for Arabic and English

* Provide:

  * complete translations (no missing strings)

---

10. QUALITY REQUIREMENTS

---

* NO mixed language UI
* NO untranslated text
* Smooth user experience

---

## DELIVERABLES

* Fully bilingual system
* Language switcher
* RTL/LTR support
* Translation files (.po / .mo)
* Verified UI consistency

---

## IMPORTANT NOTES

* Arabic must feel natural (not literal translation)
* UI must remain clean and aligned in both languages
* Ensure maintainability for future translations
