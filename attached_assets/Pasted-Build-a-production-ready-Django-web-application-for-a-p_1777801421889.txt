Build a production-ready Django web application for a private tutor to manage students, subscriptions, and an advanced smart scheduling system with financial tracking and analytics.

---

## CORE FEATURES

1. Student & Subscription Management

* Create and manage students:

  * name
  * phone
  * notes

* Each student has a subscription:

  * number of sessions per week (e.g. 3 days/week)
  * session duration (30 or 60 minutes)
  * hourly rate

* Automatically calculate:

  * weekly earnings per student
  * monthly earnings per student

---

2. Smart Scheduling System (CRITICAL)

---

Implement a robust scheduling engine:

* Calendar-based UI (day/week view similar to Google Calendar)

* Support:

  * fixed sessions
  * recurring weekly sessions

* HARD CONSTRAINTS:

  * Prevent ANY overlapping sessions (strict no-overlap)
  * Validate on backend and frontend

* Smart logic:

  * Suggest next available time slot automatically
  * Consider:

    * existing sessions
    * working hours
    * prayer time blocking
    * exceptions

---

3. Working Hours (IMPORTANT)

---

* Tutor defines working hours per day:

  * e.g. Saturday: 10:00 → 18:00

* Rules:

  * Scheduling system MUST NOT allow booking outside working hours
  * Suggestions must respect these limits

---

4. Exceptions / Overrides

---

* Tutor can define specific days as unavailable:

  * e.g. “I am not working this Tuesday”

* This overrides:

  * recurring schedules
  * working hours

---

5. Prayer Time Blocking (IMPORTANT)

---

Tutor location: Cairo, Egypt (timezone: Africa/Cairo)

* Integrate prayer times:

  * via API OR manual input

* Blocking rules:

  * Allowed: from Adhan → +10 minutes (before iqama)
  * Blocked: from +10 → +25 minutes after Adhan

* Scheduling system MUST:

  * prevent booking in blocked interval
  * adjust suggestions automatically

---

6. Attendance, Cancellation & Make-up Sessions

---

* Each session has status:

  * scheduled
  * completed
  * cancelled
  * missed

* Make-up sessions:

  * If a student misses a session:
    → they can reschedule it
  * Rules:

    * must be within the same week OR within configurable X days
  * System must track:

    * original session
    * make-up session link

---

7. Financial Reports

---

* Calculate:

  * total tutor income
  * completed sessions count
  * cancelled sessions count

* Filters:

  * date range
  * per student

---

8. Dashboard

---

Display:

* Today’s sessions
* Today’s earnings
* Alerts:

  * upcoming sessions
  * conflicts (if any)

---

9. Quick Reschedule

---

* Add action:
  → “Reschedule”

* Function:

  * Move session to nearest available valid slot
  * Respect:

    * no overlap
    * working hours
    * prayer time
    * exceptions

---

10. Analytics & Insights

---

Provide advanced analytics:

* Best working hours (most productive times)

* Most committed students

* Cancellation rate

* Occupancy rate:
  → (booked hours / available hours)

* Visualizations:

  * charts (line, bar, pie)
  * modern interactive UI

---

## TECHNICAL REQUIREMENTS

* Backend:

  * Django
  * Django ORM
  * SQLite (for simplicity, ready to scale later)

* Frontend:

  * Django Templates (DTL)
  * HTML + CSS + JavaScript
  * Modern UI (clean, responsive)

* Charts:

  * Use a JS chart library (e.g. Chart.js or similar)

* Time handling:

  * timezone-aware (Africa/Cairo)

* Architecture:

  * Clean modular structure
  * Separate scheduling logic into services layer
